# Workspace Map

## Purpose
Build Matdata Mitra — Indian election civic education assistant — for PromptWars Challenge 2 (Hack2skill + Google for Developers). Multilingual chatbot deployed on Cloud Run using Gemini API.

## Workspace Inventory
| Workspace | Folder | What lives here |
|-----------|--------|-----------------|
| App       | `app/` | Next.js 14 application (TypeScript, Tailwind, App Router). All implementation code, components, API routes, tests, Dockerfile. |
| Docs      | `docs/` | Challenge brief (`Requirments.md`), workspace organization skill reference, architectural notes, source material. |

## Naming conventions
- Dated assets:   `YYYY-MM-DD_<slug>.<ext>`
- Specs / briefs: `SPEC_<name>.md`
- Context files:  `_context.md` (one per workspace folder, read before starting any task)
- Skills:         `~/.claude/commands/<verb>-<noun>.md`

## Navigation rules
1. Read this file first.
2. Identify which workspace the task belongs to.
3. Read that workspace's `_context.md` before doing anything else.
4. For cross-workspace tasks, read all relevant `_context.md` files before starting.
5. Do not read files outside the active workspace(s) unless a routing table explicitly says to.
