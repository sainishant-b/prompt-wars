The problem statement for Challenge 2 focuses on civic education, specifically requiring participants to build an assistant that simplifies the election process for users

Key details regarding this challenge include:

Objective: Create an interactive, easy-to-follow system that explains the election process, including timelines and key procedural steps 
Approach: The goal is to break down complex electoral processes into digestible, conversational explanations so that the experience feels like talking to an informed guide rather than reading a textbook 
Adaptability: The assistant should be able to handle various user questions and adapt its explanations based on the user's existing level of knowledge 
Example Concepts: Speakers suggested several creative ideas, such as:
A phone calling agent where users can listen to explanations or ask questions about the process 
An informative agent to explain specific rights, such as what to do if a vote is proxied, citing relevant legal codes like Section 49A 
Security or monitoring solutions, such as using edge models on CCTV cameras to detect and alert management to chaos or incidents at polling booths 
Participants are encouraged to think beyond imagination and build out-of-the-box solutions that are useful and simplify the electoral experience for the general public

For the Challenge 2 submission regarding civic education and the election process, participants must adhere to the following rules and requirements:

Submission Artifacts:

Working Project: You must deploy your application using Cloud Run 
GitHub Repository: A link to your code repository is required for evaluation 
LinkedIn Post: You must document your approach, the problem statement, your thought process, and tool usage in a LinkedIn post. This post must tag Google for Developers and Hack2skill 
Program Rules:

Individual Participation: The program is for individual developers based in India 
Attempts: You are allowed three submission attempts per challenge. The final submission score is the one that counts toward the leaderboard 
Deadline: All submissions must be completed by May 3rd 
Required Tools:

Participants are required to use the Google Anti-Gravity ecosystem and Cloud Run as part of their AI-assisted development workflow
